무려 2월 5일에 초안을 썻지만 아직도 완성을 안해서 방치되었던 포스팅을 2월 23일에 고치고있다
중간에 포스팅쓰겠다고 중간중간 수정한부분들이 있었는데 초안을 전이랑 비슷하게 만들었다고 포스팅이 다 하나로 통일되었다.

slug: github-blog-first추가하면 통일 됬던 포스터가 분리가 가능하다 
물론 퍼스트는 아니지만 slug 퍼스트이기때문에 쓰기로했다

원래는 더 귀찮아서 안쓰기 전에 일단 암거나 쓰면 나중에 수정하겠지라는 마음으로 초안을 만들었는데
손도 안됨 20일 가까이 지나버렸다

원래는 자꾸 커밋하면 잔디가 안심겨서 슬퍼서 잔디 심고싶어서 찾아댕기다가 원인을 찾았다

일단 내가 내컴퓨터로 안해서 컴퓨터랑 이메일 연동을 안해서 안뜨는 줄 알았는데 포그할때마다 잔디찍히는거보면 
그건아니여서 계속 찾아봤다

그러던 중 포그한 템플릿같은거는 안될 수 있다라는 걸봤다.

그래서 일단 테스트를 했다
저 테스트 저장소를 보면 바로 잔디가 찍혀잇는게 보이는데 블로그는 안찍힌거보면
계정 잔디 시스템은 정상이고 커밋도 정상적으로 잔디 찍히는걸 볼 수 있다

# test repo 만드는 방법
1. GitHub 들어가기
오른쪽 위에 + 버튼
New repository 클릭

2. 저장소 이름 적기
예를 들어:
test-contribution

3. 설정
- Public 선택
- 아래 체크

- Add a README file 이거 꼭 체크!

- Create repository 버튼 누르기
그러면 저장소가 바로 생성

- 커밋 하나 만들어서 잔디 심기
README.md 열기

파일이 자동으로 생겨있고
오른쪽 위에 ✏️ (Edit) 버튼 누르기

내용 아무거나 추가

예:
잔디 테스트용 커밋입니다.
아래로 내려서 Commit 하기
Commit message:
test commit


그리고

- Commit directly to the main branch 선택

마지막으로

- Commit changes 버튼 클릭
- 결과는 이렇게 봐야 함

이 커밋이 내일 잔디에 찍히면 → 계정은 정상
이것도 안 찍히면 → Contribution 설정 문제

문제는 블로그 저장소(HanSeungHyo.github.io) 쪽 커밋이 잔디 집계에서 빠지는 것이라고 챗지피티가 말했다

원인은

1. Pages 저장소 커밋은 “기여”로 덜 잡히는 경우 있음

설정 파일 수정(_config.yml)
테마 파일 복붙
자동 생성 파일
GitHub에서 만든 커밋
이런 건 잔디에 누락되는 경우 많음.

2. 커밋 author/email 정보가 애매하게 기록됨

블로그 repo 커밋에는 이메일 줄이 안 떴다라는게

GitHub가 커밋은 인정하지만
Contribution graph에 연결을 안 해줌

3. GitHub Actions/Pages 배포 커밋은 잔디에 안 찍힘

블로그는 내부적으로
build
deploy
자동 작업이 섞여서
봇 커밋처럼 처리되기도 함.


그래서 잔디를 심게하는거

Settings > General > Leave fork network
원본업데이트를 못받지만 잔디는 심긴다

하나는 복사한 커밋을 붙여넣을 레포지토리를 생성해서 어찌하는건데
그냥 나는 원본업데이트를 끊었다

이유는 원본업데이트가 몇년전에 끝나기도했고 이미 완성형으로 되어있어서 따로 업데이트를 받지 않아도 된다고 생각했기때문이다.
