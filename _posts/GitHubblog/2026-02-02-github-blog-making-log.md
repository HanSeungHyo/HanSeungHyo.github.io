2년동안 저장소만 만들고 어떻게 해야될지 몰라서 방치하던 깃허브 블로그
이번에 다시 블로그를 작성해볼려고 한다.

그래서 나중가면 또 어떻게 만드는지 까먹을것 같아서
“어떻게 만들었는지”를 한 번 정리해두려고 한다.

## 시작 계기
처음엔 깃허브 만들어 놓기만하고 방치만 하다가 24년에 취업프로그램 같이 하던 친구가 깃허브 블로그를 하고 있길래 
다른 사람의 블로그만든거 보면서 저장소를 만들었다.

근데 그뒤는 알아서 블로그를 구현을 해야되다길래 거기서 막혀버려서 2년 방치했다.

그리고 친구가 잔디 좀 심어보라고해서 블로그를 다시 시작을 했다.

우선 챗지피티한데 도움을 요청하고
막히는 부분은 현재 깃허브하는 친구한테 물어봤다.

- 공부한 거 적어 좀 놓자
- 깃허브 좀 해보자

그래서 github.io 블로그를 한 번 만들어보기로 했다.

## 저장소부터 생성
제일 먼저 한 건 챗지피티 도움요청을 했다.

전에 저장소는 만들어놔서 바로 템플릿으로 갔다.

## 템플릿 가져오기

이미 아무것도 없는 상태에서 시작하다가 안했기때문에
기본 구조가 있는 템플릿을 쓰는 게 낫겠다 싶어서 템플릿을 찾았다.

사실 시작부터 챗지피티보고 막혀서 친구한테 템플릿을 추천 받았다.
https://github.com/topics/jekyll-theme

Use this template 클릭
Repository name → 내아이디.github.io
Create repository

or

Fork 클릭
Repository name → 내아이디.github.io
Create fork

파일이 많았는데 암튼 모르겠으니까 챗지피티한테 도움을 요청했다

2. 상단 메뉴에서 Settings
(코드 있는 탭들 옆에 있음)

3.왼쪽 메뉴에서 Pages
(Code and automation 쪽 아래에 있음)

4. Pages 설정
Source

Deploy from a branch 선택

Branch: main(or master)

만약 테마 뼈대에 docs 폴더가 있으면  /docs

모르겠으면 일단 / (root) 선택

- `main` 브랜치는 없고
- `master`만 있어서
- 이걸로 해도 되나 고민함

결론은 **된다**였다.
실제로 존재하는 브랜치를 선택하면 문제 없었다.

## 지금 상태
지금은:

- 블로그 주소 접속 잘 되고
- 글 하나 올리면 바로 반영되는 상태

디자인이나 꾸미는 건 아직 신경 안 쓰고,
싶지만 일단 내 블로그니까 내이름으로 바뀌긴해야겠지만
일단 글부터 쓰는 게 목표다.

## 앞으로
이 블로그는
완성된 글만 올리는 공간이라기보다는
그냥 기록용으로 쓰려고 한다.

- 만들다가 막힌 거
- 공부하다 헷갈린 거
- 나중에 다시 보고 싶은 내용

글 어떻게 쓸까 고민했는데 그냥 저렇게 쓰고
포폴용으로 나중에 다시 정리하라고 해서 이렇게 쓰고있다.
챗지피티가 계속 쓰는 게 더 중요하다고 쓰란다.

일단 만들었으니,
천천히 채워가 보자.
